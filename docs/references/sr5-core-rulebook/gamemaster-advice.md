# Gamemaster Advice

**Source:** SR5 Core Rulebook
**PDF Pages:** 334–379 (Book pages 332–377)

## Overview

This section covers run design, NPC construction, game management, campaign structures, security systems, identification, drop-in locations, reputation, lifestyles, and run rewards. The mechanical content is dense and directly implementable: tables for random run generation, NPC build methods, HTR response times, security device mechanics, SIN verification rules, lifestyle costs and options, and a formulaic reward system for cash and Karma.

---

## Rules

### Run Templates

Six canonical run archetypes define the type of job. Each template defines the primary challenge and common complications.

- **Datasteal:** Acquire information (electronic or physical) and deliver it to the employer. Primary challenge is circumventing security around the data. Data may be in transit.
- **Assassination or Destruction:** Kill a target (assassination) or irreparably damage a specific object (destruction). Primary challenge is access and plan execution; secondary challenge is escape/avoiding retaliation after the attempt.
- **Extraction or Insertion:** Extraction removes a person from a location; insertion plants a person (or object) inside one. Key difficulties: security around the target and transportation. Unwilling extraction targets add coercion complications.
- **Misdirection:** Runners serve as distraction or decoy for a separate primary operation. May involve performing what appears to be a real Datasteal or Extraction but against a false target. Runners may not know they are a misdirection.
- **Protection:** Defend a person, object, or location from attack or extraction. Key challenges: knowing who is coming, when, and what they are permitted to do in defense.
- **Delivery:** Transport an item from one location to another. Three key aspects: what is being transported, where it is now, and where it needs to go. Item properties (fragility, lethality, size, magical nature) and terrain/distance drive complications.

### Random Run Tables

Roll to generate a run on the fly. See Tables section for full data.

- **Meet Location:** Roll 1D6.

- **Twists:** Roll 1D6 to inject a complication.

### NPC Build Methods

Three methods exist for statting NPCs. All must respect absolute maximums for attributes and skills.

#### Full Build

Build the NPC exactly as a player character using the character creation rules. Assign extra Karma and cash based on experience level, spending them at PC rates. Most balanced but most time-consuming. Reserve for the most important, recurring NPCs.

#### Story Build

Assign Attributes, Skills, and equipment appropriate to the narrative role, ignoring all rules except absolute maximums. Faster than Full Build; risk of creating imbalanced NPCs. Appropriate when story demands a specific power level.

#### Improvised Build

Created on the fly during play. The GM decides how powerful the NPC is relative to the players and assigns a positive or negative modifier (e.g., −1 = slightly less capable). Look at the most similarly skilled player character; the NPC has the same skills and attributes as that player, plus or minus the assigned value. Give the NPC different skills or equipment from the player's to avoid an "evil clone."

**Example:** Group's Street Sam has Automatics 5, Agility 5. A corporate security team at −1 rolls 9 dice (10−1).

### NPC Description Components

A complete NPC has four components:

1. **Description:** 1–3 physically distinctive qualities (race, visible augmentations, distinctive dress, height/weight). Use the Physical Traits table for prompts.
2. **Personality:** At minimum a single sentence; for significant NPCs a short paragraph. Use the Personality Traits table for prompts. Avoid pure stereotypes for recurring characters.
3. **Background:** Brief history, current situation, relationships. Primarily a GM resource for consistent roleplaying and improvised plot seeds.
4. **Motivations:** Defined by two questions: _What_ does this character want? _Why_ do they want it? Motivations drive consistent behavior when players take unexpected actions.

### Game Management: Spotlight Time

Spotlight Time is measured in real-world time, not game time. Each player should receive roughly equal Spotlight Time per session. Techniques:

- Know which players are extroverted (grab spotlight naturally) vs. introverted (need prompting).
- Cut away from one player's action to give another a turn; return later.
- Keep scene notes modular so focus can be shifted to different characters.

### Game Management: Pace

The GM watches real-time engagement levels. If players are bored, cut scenes short and advance the plot. If players are highly engaged, do not interrupt. Controlling pace means adjusting timing of events to maximize player engagement.

### Handling Surprises: Key Plot Points

When players take unexpected actions, identify the **key plot points** — the irreducible story elements. Everything else can be changed. Three adaptation strategies:

- **Relocate:** Move the key plot point to wherever the players chose to go, changing location, character, or delivery method. Most common technique.
- **Replace:** When no part of the original scene remains viable, improvise an entirely new scene that still delivers the key plot point.
- **Remove:** When the story develops such that the key plot point no longer makes sense, remove it entirely. Use for pacing purposes.

### Campaign Time and Character Advancement

The GM sets the in-game time between runs (every month, every six weeks, etc.), which determines advancement pace. The recommended baseline: players should be able to raise a minor skill or buy a minor piece of equipment every run; raise a major skill or buy a major piece of equipment every three runs. Adjust run rewards if runs happen at a different pace than planned.

### Alternate Campaign: Street Scum

Characters are not professional shadowrunners. During character creation, reduce Attribute Points, Skill Points, and Resources. Instead of choosing each row of the Priority table once, characters should **select once from row B, once from row C, once from row D, and twice from row E, or twice each from rows C and D and once from row E.**

### Alternate Campaign: High Life

Characters are elite-tier shadowrunners with top-tier resources. During character creation, increase Attribute Points, Skill Points, and Resources above default. Using the Priority Table, **players select twice from row A, once from row B, and twice from row C. No Availability limits apply.**

### Alternate Campaign: Military or Mercenary

Character creation is close to standard rules with Attribute Points and Skill Points at normal values. **Resources should be tripled or more, with Availability limits on military equipment ignored.** Alternatively, the GM may ignore equipment purchase entirely and assign a pool of equipment available to the team.

### Alternate Campaign: DocWagon

Character creation is close to standard. Resources should be selected from a limited GM-determined list or assigned based on role.

### Alternate Campaign: Criminal Syndicates

Character creation amount of points remains the same as standard. Skill selection may differ. GMs may restrict equipment Availability but career criminal syndicate members generally have access equivalent to professional shadowrunners.

### High Threat Response (HTR)

HTR teams are backup/response forces deployed when a primary threat is identified. They are roughly equivalent to professional runner teams in training and equipment. They may be internal security or contracted from firms such as Lone Star, Knight Errant, or Eagle Security.

**HTR response time is determined by the security level of the location.** See the HTR Team Response Time table.

### Countering Physical Threats: Cover

Security personnel will always seek cover at the start of a gunfight (see **Taking Cover**, p. 166). If no cover exists, they withdraw to find it. Cover can be created with smoke grenades (p. 435), flash bangs (p. 435), or magical effects (p. 289). Security using vision enhancement will employ lighting control (darkness) as cover-equivalent.

### Countering Physical Threats: Suppression

Teams use covering fire (a Suppression attack, p. 179) to pin enemies while repositioning. One team member fires Suppression while others change position.

### Countering Physical Threats: Ambushes

Security knows local terrain, chokepoints, and designated ambush positions. Key element is Surprise (see **Surprise**, p. 192). Security will use terrain to maximize cover advantage and put attackers in disadvantageous positions.

### Traps

A trap has two elements: **trigger** and **response**.

- **Trigger:** The sensor or event that activates the trap. Can be a trip line, pressure plate, electronic sensor, biometrics, or RFID scanner. Triggers must also be designed to _not_ fire for authorized personnel.
- **Response:** What happens when the trap fires. Ranges from locking doors and containing intruders to releasing gas or deploying monofilament trip lines or automated gun drones. Specific devices listed under Security Devices (p. 362).

### Countering Matrix Threats: Access Limitation

By default all devices run in normal mode (broadcasting presence). Security devices run silent (p. 235) to prevent detection. Network administrators increase the Noise rating for anyone in the vicinity. Physical methods (earthworks, vegetation, Faraday cages, wireless-inhibiting wallpaper) also increase Noise between devices and attackers.

### Countering Matrix Threats: PANs and WANs

Slaving less-capable devices to a more powerful Master device (PAN, p. 233; WAN, p. 233) causes all slaved devices to use the Master's Rating for tests instead of their own. Direct connection (physical/wired) allows a hacker to target a slave device separately from its Master, bypassing WAN protection; prevent this by physically protecting the device.

### Countering Matrix Threats: Hosts and IC

A Host (p. 359) acts as Master for all devices within it and runs IC programs. IC can be activated in a single Combat Turn. A Host can have up to its Rating in active IC programs at one time. IC is crashable but can be reactivated the next turn. The only continuously running IC is a **Patrol IC**, which monitors for intrusion like a physical guard.

### Countering Matrix Threats: GODs and Spiders

- **Spider:** A metahuman security professional who oversees a network. Has same skills/equipment as a hacker but defends rather than attacks. Can command any device in their system, launch/shut IC, raise/cancel alarms, or perform emergency shutdown. Can be on-site or remote.
- **GOD (Grid Overwatch Division):** Corporate Court entity. Responds to serious or repeated security breaches. A GOD agent ("G-Man") has the equivalent of an owner mark on any device on their grid.
- **demiGOD:** Branch-level GOD agents; best-equipped Matrix security forces available.

### Countering Matrix Threats: Wired Security

Forgoing wireless entirely; rare due to inconvenience. All Matrix devices connect via wires. Still vulnerable if a wire is breached and tapped. Encountered only from exceedingly security-paranoid organizations.

### Device Ratings

Standard device Ratings for Matrix interaction. See Device Ratings table.

### Countering Magic Threats: Tactics for Mundanes

Primary tactic: **target priority** — eliminate the magician first. "Geek the mage first." Wounding a magician reduces spell effectiveness. Secondary tactic: **limiting visibility** — a magician cannot cast spells against things they cannot see. Use cover, flash bangs, smoke, or lighting changes.

### Contracted Magical Security

Most organizations cannot afford on-site Awakened personnel 24/7. Magical security is typically "on call." When a magical threat is identified:

1. Magic backup is called in.
2. First response: astral scouting to confirm the threat.
3. If confirmed, the magician may summon a spirit or call a bound spirit, then stay on astral or return to physical body depending on contract level.
4. Often arrives as part of an HTR team.

### Magical Barriers

Passive magical protection. Can be established on the astral plane (blocking astral observation/projection) or the physical plane (adding protection on the physical side). The magician who created the barrier is notified if it is breached, attacked, or destroyed.

### Critters and Spirits in Security Roles

- **Spirits:** Summoned and given a Task using one of their Services to protect an area. Must be given simple, specific instructions. Are bound for the security role. Can engage astral entities, report astral activity, patrol restricted areas.
- **Awakened Critters:** Permanent once deployed; no Service limit. Require lengthy and expensive training. Can perceive astral space and physical space simultaneously. Can engage astral entities. Examples: Barghest (p. 403), Basilisk (p. 403), Hellhound (p. 405).
- **Drones:** Immune to spells targeting organic/sentient beings. Resist physical magical spells at significant cost to the caster. Cheap to deploy at scale. Vulnerable to hacking. Most popular counter to magical threats.

### Security Devices: Landscaping

Trees, shrubs, hills, gullies, ponds, and lakes used as barriers and obstacles. Dense brush applies an appropriate modifier to Infiltration Tests. Foliage also increases Noise (p. 230) for wireless devices on the other side.

### Security Devices: Barriers (Fencing)

Wooden, stone, chain-link, and electrified fences. Barbed/concertina wire and monowire on top. Climbing past fencing requires a Climbing + Agility (3) Test; failure causes damage per the Fencing Table. Walls follow standard Barriers rules (p. 197); HVAC systems within walls can increase Armor rating (p. 197).

**Fencing Table:**

| Difficulty  | Threshold | Damage |
| ----------- | --------- | ------ |
| Barbed      | 1         | 4P     |
| Concertina  | 1         | 5P     |
| Monowire    | 3         | 8P     |
| Electrified | 2         | 6S     |

### Security Devices: Doors, Windows, and Locks

- **Key locks:** Defeated with Locksmith + Agility [Physical] (Lock Rating, 1 Combat Turn) Extended Test. Autopickers add their Rating to the dice pool.
- **Transponder-embedded keys:** Require Hardware + Logic [Mental] (Lock Rating, 1 minute) Extended Test. If the same character is picking and calibrating, apply −2 dice pool modifier.
- **Maglocks:** Widespread in 2075. "Keys" can be physical (keypad, swipe card, proximity card, memory string), biometric, or any combination. Monitored by security hacker/rigger. To bypass: remove case with Locksmith + Agility [Physical] (Maglock Rating x 2, 1 Combat Turn) Extended Test. Case can be smashed/shot (Barrier rating = maglock rating). Anti-tamper systems rated 1–4 require an additional Locksmith + Agility [Physical] (anti-tamper system rating) Test; failure triggers alarm.
- **Keypads:** Rewire circuits — Locksmith + Agility [Physical] (Maglock Rating x 2, 1 Combat Turn) Extended Test. A maglock sequencer (p. 448) makes an Opposed Test vs. maglock rating; sequencer wins = maglock opens (case must be opened first).
- **Cardreaders:** Defeated same as keypads. Maglock passkeys (p. 448) don't require opening the case. If a valid keycard is acquired, copy it with a keycard copier (p. 447) — make an Opposed Test between the passkey/forged keycard rating and maglock rating.
- **Print scanners (fingerprint/palm/retinal/DNA):** Multiple defeat methods. Synthetic print sleeve for fingerprints/thumbprints (cellular glove molder requires authorized print to copy, p. 447). Retinal duplication cybereye accessory (p. 453). Fake print: Opposed Test between duplicate and maglock rating.
- **Voice recognition:** Oppose with voice recording or Voice Modulator cyberware (p. 452). Opposed Test: voice recognition system vs. bypass equipment; most hits wins.
- **Facial recognition:** Opposed Test — Disguise + Intuition [Mental] vs. Device Rating. +2 dice pool modifier if picking the disguised character out of a crowd.
- **Breath/cellular/DNA scanners:** Enzyme bath synthesized with Chemistry + Logic (5, 1 hour) Extended Test.
- **MADs (Magnetic Anomaly Detectors):** Detect ferrous-metal weapons/objects. Device Rating as dice pool; threshold 1 for a single hit detecting any ferrous-metal weapons or objects.
- **Millimeter wave/cyberware scanners:** Cyberware Scanner Table thresholds. Standard cyberware/weapons threshold 1; Alphaware/other items 2; Betaware 3; Deltaware 5+. Situational modifiers: 2+ implants/items +1; 4+ implants/items +2; 6+ implants/items +3.

### Security Devices: Sensors and Scanners

- **Lighting:** Hardware + Logic [Mental] (5, 1 Combat Turn) Extended Test to subvert most switches.
- **Alarms:** Electrical circuit-based (door/window opens = circuit breaks = alarm). Bypass: Hardware + Logic [Mental] (5, 1 minute) Extended Test to fool electrical contacts.
- **Wire (trip wire):** Perception Test to notice; triggers alarm or damage (use Fencing Table for damage wires).
- **Trip beams:** Perception + Intuition [Mental] (2) Test for visible beams; threshold 3 for infrared. Reduced to threshold 1 in smoke/aerosol spray. Squeezing past: Escape Artist + Agility [Physical] Test vs. GM-determined threshold.
- **Pressure pads:** Perception Test threshold 3. If a character steps on a pad, threshold drops to 1 (but usually too late). After stepping on pad: second Perception Test (threshold 1) — if successful, Reaction + Intuition (3) Test to remove pressure before weight exceeds allowance; Body serves as negative dice pool modifier.
- **Pressure mesh:** Perception threshold 4. More sensitive than pads after stepping on: threshold 3 for second test.
- **Motion sensors:** Transmit an ultrasonic field. Intruders may detect ultrasound sensor in passive mode within 5 meters. Defeating: move no faster than one-half meter per Combat Turn; Sneaking + Agility [Physical] (3) Test. Extra Initiative Dice (cybernetics) apply a negative dice pool modifier equal to the extra dice.
- **Proximity wire (capacitance wire):** Detects electrical charge of a metahuman body within 2 meters. Triggers regular alarm or switches on cameras/other measures.
- **Sound/vibration detectors:** Infiltration + Agility [Physical] (3) Test to sneak past a known sound detector.
- **Security cameras:** Visual, low-light, infrared, ultraviolet types. Infrared picks up body heat signatures (foiled by Improved Invisibility). Thermographic cameras can detect around corners via reflective surfaces. Low-light sensors amplified by ambient light (overpowered by bright light; flash-bang grenades, p. 435 exploit this). Spotting cameras: Perception Test. Hidden camera: apply Perception Thresholds Table (p. 136) modifier.
- **Olfactory/chemical scanners (chemsniffer):** Roll a dice pool equal to Device Rating against threshold 2 (3 if explosives/ammo are hermetically sealed). See Chemical Detection Modifiers table. Also function as pheromone scanners: Device Rating vs. threshold 3 (2 for characters with tailored pheromone bioware).
- **Automated gun systems:** Weapon-mounted drones at fixed locations (180-degree arc) or slide-mounted tracks. Use basic sensors and Targeting autosofts; follow standard drone rules (p. 269).
- **Containment systems:** When alarm triggers, shutters drop over windows and doors, sliding walls/gates activate, laser/monowire mazes and radio jamming deploy. Fills area to detain intruders.
- **Gas delivery systems:** Dispersal fills 30 cubic meters in one Combat Turn. GM secretly conducts Perception Tests to notice gas (threshold based on how noticeable the gas is — many are colorless/odorless). Olfactory scanner (p. 452) may alert equipped characters. See p. 408 for gas types.
- **Marking systems:** UV dye, RFID tags, DNA-encoded material, or nanite tags sprayed unobtrusively over exits and traffic areas.

### System Identification Number (SIN)

A SIN is issued by a country or AA/AAA extraterritorial corporation at birth (or change of citizenship). It is a string of characters generated from vital statistics via a mathematical algorithm guaranteeing uniqueness. A SIN is linked to: name, birth date, place of birth, issuing nation, DNA, retinal scan, and fingerprints. Registered with two master databases: the issuing country's database and the **Global SIN Registry (GSINR)** controlled by the Corporate Court.

### Issuing a SIN

Issued at birth at a hospital, clinic, or at home with assistance of a Renraku DulaDrone. A new SIN is issued on change of citizenship.

### Licenses

Specific permissions required to legally own/use restricted items or perform restricted activities. Acquiring a license involves application, fees, SIN check, and possibly certification/testing/training. Exact details are gamemaster-determined. See License Examples table for common types.

### Fake SINs

Fake SIN quality is represented by its Rating (1–6). See Fake SIN Details table. Higher Rating = more complete biographical and biometric data. Rating 5+ SINs include valid biometrics for a real person (with samples). For availability and prices of purchasing fake SINs, see p. 367 (in gear chapter).

> **Cross-reference:** Fake SIN availability and cost are in the gear chapter, p. 367.

### Checking a Fake SIN

The sophistication of the verification system (Device Rating) determines how deeply a fake SIN is checked. Make a **Simple Device Rating x 2 Test** with a threshold equal to the Rating of the fake SIN (use Device Ratings, p. 234 for SIN verification system ratings).

- **Under threshold:** System reports no problem.
- **Threshold reached but not exceeded:** System reports something "odd"; operator may investigate further.
- **Threshold exceeded:** System reports the SIN as false; may immediately notify authorities. Fake SIN is now **burned**.

See SIN Verification Details table for what each Rating level checks.

### Burned SINs

When a fake SIN is found to be false, the SIN Verification System notes it immediately and (if Matrix-connected) sends a special emergency message back to the SIN registry of the country where verification occurred. Any further use of that fake SIN in that country automatically fails. The SIN is only burned in the country where it was detected — data sharing between countries is imperfect.

> **Ambiguity:** The book states data sharing between countries "is not always perfect or instantaneous." At the GM's discretion, a SIN burned in one country may not be burned in another.

### Reputation: Street Cred

Street Cred represents lifetime accomplishments. Formula: **Street Cred = Total Karma earned ÷ 10 (rounded down).** Additional Street Cred may be awarded by the GM for particularly noteworthy accomplishments. Street Cred applies as a positive **Limit Modifier** to all Social Tests in situations where the character's reputation would be known.

> **Cross-reference:** Street Cred can be sacrificed to reduce Notoriety: permanently sacrifice 2 points of Street Cred to reduce Notoriety by 1.

### Reputation: Notoriety

Notoriety represents negative reputation. **Starting Notoriety = 0, modified by qualities: +1 per negative quality, −1 per positive quality** (Positive: Blandness, First Impression, Lucky; Negative: Addiction, Bad Luck, Combat Paralysis, Elf Poser, Gremlins, Incompetent (any skill), Ork Poser, Scorched, SINner (criminal SIN only), Spirit Bane, Uncouth, Uneducated, Weak Immune System).

During play, the GM awards Notoriety for:

- Refusing to finish a run
- Insulting or pissing off a Mr. Johnson
- Incredibly obnoxious or callous behavior
- Betraying team members
- Killing innocents
- Earning a powerful enemy
- Knowingly working for a dragon
- Exceptional bad luck that leads to disaster

Characters cannot earn Notoriety for something they already have a point for. High Notoriety makes contacts and others less willing to help. **Reduce Notoriety:** permanently sacrifice 2 points of Street Cred per 1 point of Notoriety.

### Reputation: Public Awareness

Public Awareness represents how well the character is known to the media, authorities, and general public. The GM awards Public Awareness for:

- Fighting with law enforcement or HTR
- Extracting a well-known person
- Blowing something up in a populated area
- Leaving significant physical evidence
- Being seen by many eyewitnesses
- Broadcasting/posting videos of crimes on Matrix fansite

See Public Awareness Table for what each rating level means. Public Awareness has no direct mechanical benefit — it is a liability that attracts unwanted attention.

### Lifestyles

Characters must select a primary Lifestyle. Additional lifestyles may be maintained at full cost (for safehouses, alternate identities, etc.).

| Lifestyle    | Cost per Month                                 |
| ------------ | ---------------------------------------------- |
| Luxury       | 100,000¥ and up                                |
| High         | 10,000¥                                        |
| Middle       | 5,000¥                                         |
| Low          | 2,000¥                                         |
| Squatter     | 500¥                                           |
| Streets      | Free                                           |
| Hospitalized | 500¥/day basic care; 1,000¥/day intensive care |

**Hospitalized** is a special lifestyle that applies only when a character is sick or injured. Characters cannot own this lifestyle — they only pay for it until they get well or go broke.

### Lifestyle Options

Options can be selected for any lifestyle except Streets or Hospitalized. The GM may disallow any option if it would not be plausible for the character's described residence or if there is no chance the bonus/penalty ever comes into play.

| Option                    | Cost Modifier     | Effect                                                                                                                 |
| ------------------------- | ----------------- | ---------------------------------------------------------------------------------------------------------------------- |
| Special Work Area         | +1,000¥/month     | Workshop/garage/office/studio. Skill checks relevant to the setting have their Limit increased by 2.                   |
| Extra Secure              | +20% of lifestyle | HTR response (p. 356) and other security response rolls made one level better than the area would normally be.         |
| Obscure/Difficult to Find | +10% of lifestyle | Any Sneaking Skill checks (owner of property excluded) made in the vicinity of the residence are made with −2 penalty. |
| Cramped                   | −10% of lifestyle | Logic-linked Skill checks have their Limit reduced by 2 (minimum 1) when performed in this space.                      |
| Dangerous Area            | −20% of lifestyle | Any HTR response or similar security response rolls are made as if this area were one level lower than it actually is. |

### Paying the Bills

Monthly lifestyle costs must be paid. Prepayments can be made for any number of months; landlords are unlikely to refund money.

**Missed payment mechanic:**

- Each time a payment is missed, roll 1D6.
- If result is **greater than** the number of consecutive months missed: landlord has let the character slide; payment must still be made up later.
- If result is **less than or equal to** the number of months missed: character is out the door.

If this is not the primary lifestyle, the unit is lost. If the character missed payments on their **primary** lifestyle: **downgrade the lifestyle one level.** Character may also owe back rent to a bank, loan company, or loan shark.

### Buying a Lifestyle

A character can permanently buy a lifestyle by paying **100 months' worth of upkeep** (e.g., Luxury = 10,000,000¥). This represents investments, trust funds, etc. A permanent lifestyle can be lost through enemy action or sheer bad luck.

**Selling a permanent lifestyle:** If the character has a couple of months to broker, roll 2D6 and multiply by 10% to determine what percentage of purchase price they receive. If the character does not have the SINner quality (p. 84), roll only 1D6. Also roll 1D6 if the character must dump the home/investments fast or through an agent because they are on the run.

### Team Lifestyles

A team can buy a joint lifestyle. Cost is an extra 10% per additional person. For Low Lifestyle or higher, one member must be the **tenant of record** — the one legally stuck with the debt if the team fails to pay.

### Run Rewards: Cash

Base formula per player:

1. **Base cost:** 3,000¥
2. **Highest opposing dice pool ÷ 4 (rounded down)** = multiplier part 1. The opposing dice pool must be from an Active Skill used directly against the players (not a Knowledge Skill). Use permanent enhancements only (no equipment bonuses, conditional/situational modifiers). Divide total dice pool by 4.
3. Add additional multiplier values from the Cash Rewards table (see table for full situational modifiers).
4. Apply **cost modifiers** (percentage adjustments to the final amount):
   - Standard run: 0%
   - Cold-hearted bastard run (wetwork, helping corps oppress people, drug running, human trafficking): +10–20%
   - Good feelings run (hooding, helping the little guy): −10–20%

**Base cost may also be increased by 100¥ per net hit on a Negotiation Test at the start of the run.**

### Run Rewards: Karma

Karma is awarded to each player after a run is completed. Add up applicable situations from the Karma Rewards table, then apply a Karma Modifier.

**Karma Rewards:**

- Character survived: 2
- Group completed all objectives: 2
- Group completed some objectives: 1
- Overall adventure challenge: Highest opposed Dice Pool ÷ 6 (rounded down)

**Karma Modifiers:**

- Standard run: 0
- Cold-hearted bastard run: −2
- Good feelings run: +2

## Implementation Notes

- **Street Cred as Limit Modifier:** Street Cred applies as a positive Limit Modifier to Social Tests where the character's reputation would be known. This is distinct from a dice pool bonus — it raises the maximum successes that can be applied.
- **Reputation system has three independent scores:** Street Cred, Notoriety, and Public Awareness are tracked separately. None directly affects the others except the voluntary Street Cred sacrifice mechanic.
- **Notoriety quality initialization:** On character creation, loop through all qualities and apply +1 for each negative quality (from the Notoriety negative list) and −1 for each positive quality (Blandness, First Impression, Lucky). Clamp at 0 minimum is not specified — starting Notoriety can theoretically go below 0 if a character has multiple positive qualities, but this is rarely relevant in practice.
- **Lifestyle option applicability:** The GM (or game logic) must gate lifestyle options by plausibility. In software terms, the option list should be filterable by whether the option can logically apply to the lifestyle level and residence type.
- **Run reward calculation is per-player, not per-group:** Both cash and Karma are awarded to each player individually. The cash base (3,000¥) and Karma awards are per-player values.
- **Highest opposing dice pool selection:** Only Active Skills used directly against the players count. Exclude: Knowledge Skills, equipment bonuses, conditional/situational modifiers. Include: permanent cyberware/magic enhancements to the skill or attribute.
- **HTR security level Z:** "Redmond Barrens, Chicago Containment Zone" = 2D6 hours, meaning HTR response may effectively never arrive for practical purposes.
- **Burned SIN geography:** Implement as a per-country (or per-jurisdiction) flag on the SIN record, not a global flag.
- **Fake SIN Rating 5 biometrics:** Rating 5 SINs contain "valid biometrics for another person (with samples)" — meaning organic samples (blood, skin cells, hair) must have been obtained. This is flavor but informs why high-Rating fake SINs are expensive and rare.
- **Alternate campaign priority table modifications** are character creation rules that override the standard Priority table selection rules. These should be implemented as campaign-level configuration that modifies the character creation flow.
- **Drop-in locations** are flavor/GM tools; the security profiles are not mechanically precise — they describe what security elements are typically present, not stat blocks.
