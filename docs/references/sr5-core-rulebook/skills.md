# Skills

**Source:** SR5 Core Rulebook
**Book Pages:** 128–155
**PDF Pages:** 130–157

---

## Overview

Skills govern everything a character does in Shadowrun: physical action, social manipulation, technical work, magic, Matrix operations, and vehicle handling. Every skill has a numerical Rating (1–12, or 13 with the Aptitude quality) and a linked attribute. Dice pools are formed by adding Skill Rating + linked Attribute Rating. Skills are divided into three broad categories: Active, Knowledge, and Language. Active skills are further subdivided into Combat, Physical, Social, Magical, Resonance, Technical, and Vehicle groupings.

Located: "Skills" — PDF pages 130–157 (Book pages 128–155)

## Rules

### Skill Ratings

Skill Rating is a number from 1 to 12 (maximum 13 with the Aptitude quality, p. 72). Rating 6 is the maximum for starting player characters. Rating 0 means untrained (the character has general awareness but no formal training and can attempt defaultable skills at –1 die). A character with "No Rating" (below 0, achievable only through the Incompetent negative quality or a defined character history deficiency) cannot default on the skill and is treated as completely unaware of it.

**Skill Rating descriptors:**

- 0: Untrained
- 1: Beginner
- 2: Novice
- 3: Competent
- 4: Proficient
- 5: Skilled
- 6: Professional (starting PC maximum)
- 7: Veteran
- 8: Expert
- 9: Exceptional
- 10: Elite
- 11: Legendary
- 12: Apex
- 13: Apex (Aptitude quality required)

### Linked Attributes

Each Active skill has a single linked attribute (the attribute most closely associated with the skill). The standard dice pool is `Skill Rating + Linked Attribute`. A full list of Active skills by linked attribute appears on pp. 151–153 of the source book (see `skills.json` table `active-skill-list`).

### Skill Types

Three categories:

- **Active Skills:** Used to take action and affect the world. Linked to a single attribute. Used in tests.
- **Knowledge Skills:** Represent what a character knows. Linked to Logic or Intuition depending on subcategory. Do not add dice to Active skill tests; provide background context and enable legwork.
- **Language Skills:** Represent languages a character can speak, read, and write. Linked to Intuition. Higher rating = more fluency.

### Skill Groups

A skill group is a collection of 3–4 related skills bought at a single rating. Buying a skill group at Rating X is equivalent to having each member skill at Rating X, and is cheaper than buying them individually.

**Breaking a skill group:** If you advance one skill in a group without advancing the rest, the group is broken. You receive each skill at the former group rating, then may advance individual skills independently. To reconstitute the group, all member skills must first be raised back to the same rating.

> **Cross-reference:** Character Advancement, p. 103, for karma costs of skill groups.

### Specializations

A specialization is a focused subset of a skill. It adds +2 dice to any test where the specialization applies. Written as `Skill Name (Specialization) Rating (+2)` on the character sheet (e.g., `Firearms (Pistols) 4 (+2)`).

- A character may have multiple specializations within the same skill, but only one applies at a time.
- New characters may take one specialization per skill at character creation (p. 62).
- You must have Rating 1 or higher in a skill before taking a specialization.
- Specializations are not allowed for skill groups.

### Defaulting

When a character attempts a skill test without having the skill, they may **default** if the skill permits it (defaultable skills are not italicized in the Active Skill List on p. 151). Defaulting uses only the linked attribute for the dice pool and imposes a **–1 dice pool modifier** in addition to any other situational modifiers.

Skills that cannot be defaulted are listed in italics in the Active Skill List. Attempting them without any rating results in automatic failure — the character simply cannot try.

Edge can be used to Push the Limit on a defaulting test (p. 56).

### Substituting Skills

The gamemaster may allow one skill to substitute for another when a character lacks the required skill, typically with a dice pool penalty (e.g., –3). Active skills can substitute for Knowledge skills (not vice versa) — knowing the mechanics of pistols doesn't make you a better shot.

### Dice Pool Construction

Dice pool = Skill Rating + Linked Attribute Rating + any applicable modifiers (specialization bonus, situational modifiers, wound modifiers, etc.).

## Combat Active Skills

All Combat Active skills are linked to **Agility** unless otherwise noted. For combat application rules see the Combat chapter, p. 158.

### Archery (Agility)

Used to fire string-loaded projectile weapons (bows, crossbows, slingshots).

- Default: Yes
- Skill Group: None
- Specializations: Bow, Crossbow, Non-Standard Ammunition, Slingshot

### Automatics (Agility)

Covers submachine guns and fully automatic carbines — firearms larger than handheld pistols but smaller than rifles.

- Default: Yes
- Skill Group: Firearms
- Specializations: Assault Rifles, Cyber-Implant, Machine Pistols, Submachine Guns

### Blades (Agility)

All handheld slashing and stabbing weapons: daggers, swords, axes.

- Default: Yes
- Skill Group: Close Combat
- Specializations: Axes, Knives, Swords, Parrying

### Clubs (Agility)

All handheld bludgeoning instruments: baseball bats, crutches, maces, staves.

- Default: Yes
- Skill Group: Close Combat
- Specializations: Batons, Hammers, Saps, Staves, Parrying

### Exotic Ranged Weapon (Specific) (Agility)

Must be taken once per specific unusual ranged weapon (blowguns, gyrojet pistols, flamethrowers, lasers). Cannot default. No specializations.

- Default: No
- Skill Group: None
- Specializations: None

### Heavy Weapons (Agility)

All projectile weaponry larger than an assault rifle: grenade launchers, machine guns, assault cannons. Handheld/non-vehicle-mounted only — use Gunnery for vehicle-mounted weapons.

- Default: Yes
- Skill Group: None
- Specializations: Assault Cannons, Grenade Launchers, Guided Missiles, Machine Guns, Rocket Launchers

### Longarms (Agility)

Extended-barrel weapons: sporting rifles, sniper rifles, shotguns (braced against shoulder).

- Default: Yes
- Skill Group: Firearms
- Specializations: Extended-Range Shots, Long-Range Shots, Shotguns, Sniper Rifles

### Pistols (Agility)

All handheld pistols: tasers, single-shots, semi-automatics, revolvers.

- Default: Yes
- Skill Group: Firearms
- Specializations: Holdouts, Revolvers, Semi-Automatics, Tasers

### Throwing Weapons (Agility)

Any handheld item thrown as a weapon.

- Default: Yes
- Skill Group: None
- Specializations: Aerodynamic, Blades, Non-Aerodynamic

### Unarmed Combat (Agility)

Self-defense and attack using the body as a weapon. Includes martial arts and cybernetic implant weaponry.

- Default: Yes
- Skill Group: Close Combat
- Specializations: Blocking, Cyber Implants, Subduing Combat, or by specific Martial Art

## Physical Active Skills

Physical skills govern actions taken with the body (excluding combat). Linked attributes are listed with each skill.

### Disguise (Intuition)

Non-magical identity masking including makeup and enhancement. See Using Disguise and Impersonation, p. 136.

- Default: Yes
- Skill Group: Stealth
- Specializations: Camouflage, Cosmetic, Theatrical, Trideo & Video

**Using Disguise and Impersonation:**

- When makeup/costuming is first applied: `Disguise + Intuition [Mental]` test. Hits become the threshold for attempts to see through the disguise.
- Impersonation can improve a disguise or stand alone: `Impersonation + Charisma [Social]`. Hits add to the threshold for breaking the disguise (cumulative with Disguise hits).
- If Impersonation is used without a disguise (vocal only), treat as a standard Opposed Test.
- A Disguise Kit (p. 443) can be used to build a disguise using Building & Repairing rules (p. 146). Add half the Extended Test threshold as a dice pool bonus to the Disguise Test, maximum +4.
- Opposed: `Perception + Intuition [Mental]`

### Diving (Body)

Actions performed underwater: diving, swimming underwater, complex diving equipment, holding breath.

- Default: Yes
- Skill Group: None
- Specializations: By breathing apparatus (Liquid Breathing Apparatus, Mixed Gas, Oxygen Extraction, SCUBA), by condition (Arctic, Cave, Commercial, Military, etc.), Controlled Hyperventilation

### Escape Artist (Agility)

Escaping from bindings using body contortion and manual dexterity. Test: `Escape Artist + Agility [Physical]` vs. threshold from the Escape Artist Table. Success frees the character for 1 minute (20 Combat Turns) divided by net hits.

**Escape Artist Table thresholds:**

- Rope, plastic cuffs: 2
- Metal handcuffs: 3
- Straitjacket: 4
- Containment manacles: 5
- Reinforced material: +1 (modifier, not base threshold)

**Situation modifiers:**

- Character being watched: –2
- Character physically restrained: –(Restrainer's Strength)
- Character has picks or cutting implement: +2

- Default: Yes
- Skill Group: None
- Specializations: By restraint (Cuffs, Ropes, Zip Ties, etc.), Contortionism

### Free-Fall (Body)

Any jump from height including parachuting, wingsuit flying, BASE jumping, bungee, zipline.

- Default: Yes
- Skill Group: None
- Specializations: BASE Jumping, Break-Fall, Bungee, HALO, Low Altitude, Parachute, Static Line, Wingsuit, Zipline

### Gymnastics (Agility)

Balance, general athleticism, use of body. Governs climbing and jumping with special rules. See Using Gymnastics, p. 134.

- Default: Yes
- Skill Group: Athletics
- Specializations: Balance, Climbing, Dance, Leaping, Parkour, Rolling

**Climbing (Gymnastics + Strength [Physical]):**

- Complex Action. Hits determine vertical/lateral movement per the Climbing Table.
- Assisted climbing (with gear): easier with +2 pool modifier.
- Surface difficulty modifiers apply (see `climbing-table` in skills.json).
- A failed test while climbing/rappelling: must make `Reaction + Strength` to hold on. Failure = start falling (~20 m/Combat Turn). Character may attempt to stop by making `Reaction + Strength [–2]` on next Action Phase.
- Characters with climbing gear who fall may have safety lines checked: `Free-Fall + Logic [Mental]` vs. threshold equal to half falling character's Body (round down). Success = dangling; failure = Falling Damage (p. 172).

**Rappelling:**

- Gear-aided descent. Requires `Free-Fall + Body [Physical] (2)` Success Test to set up.
- Falls 20 meters per Combat Turn. Each net hit beyond threshold increases descent by 1 meter that Combat Turn.
- Taking another Simple Action during same phase (e.g., firing): –2 dice pool to both actions.
- Stopping requires another Free-Fall Test at same threshold.

**Jumping (Gymnastics + Agility):**

- Running leap: 1 meter horizontal per hit; maximum = Agility × 1.5 meters.
- Standing jump: same test, same cap.
- Vertical jump: 0.5 meters altitude per hit; maximum = 1.5 × height.

### Palming (Agility)

Sleight-of-hand: snag, hide, and pass off small objects. Cannot default.

- Default: No
- Skill Group: Stealth
- Specializations: Legerdemain, Pickpocket, Pilfering

### Perception (Intuition)

Spotting anomalies using any sense. Test: `Perception + Intuition [Mental]`.

- Can be taken as Simple Action (Observe in Detail).
- Net hits determine detail level.
- Vision-based tests use Visibility and Light modifiers from Environmental Modifiers (p. 173).
- Opposed by Sneaking (Using Stealth Skills, p. 136). GM may substitute Perception for Sneak in Stealth group.
- Teamwork Test rules apply when multiple characters search for the same thing (p. 49).

**Perception Test Modifiers:**

- Perceiver is distracted: –2
- Specifically looking/listening for it: +3
- Object/sound not in immediate vicinity: –2
- Object/sound far away: –3
- Object/sound stands out: +2
- Interfering sight/odor/sound: –2
- Perceiver has active enhancements: +(Rating)
- Visibility and Light: Environmental Factors, p. 173

**Perception Thresholds:**

- Obvious/Large/Loud: 1
- Normal: 2
- Obscured/Small/Muffled: 3
- Hidden/Micro/Silent: 4

- Default: Yes
- Skill Group: None
- Specializations: Hearing, Scent, Searching, Taste, Touch, Visual

### Running (Strength)

Increases distance covered in a Combat Turn via Sprint Action (p. 162). Test: `Running + Strength [Physical]`. Each hit adds 1 or 2 meters (metatype-dependent) to movement for that Combat Turn. Maximum non-stop running = (Body + Running) × 10 minutes; Fatigue Damage applies after (p. 172).

- Default: Yes
- Skill Group: Athletics
- Specializations: Distance, Sprinting, by terrain (Desert, Urban, Wilderness, etc.)

### Sneaking (Agility)

Remaining inconspicuous and hidden. Opposed Test: `Sneaking + Agility` vs. `Perception + Intuition`. GM may allow substitution of a Stealth group skill for Perception at no penalty. See Using Stealth Skills, p. 136.

- Default: Yes
- Skill Group: Stealth
- Specializations: Location type (Jungle, Urban, Desert, etc.)

### Survival (Willpower)

Staying alive in extreme environmental conditions. Test: `Survival + Willpower [Mental]`. Failure = no food/shelter found and Stun damage equal to twice the threshold (unresistable, cannot heal without sleep). If Stun Monitor overflows, character falls unconscious and keeps taking Physical damage until rescued or dead.

Exposure for 24+ hours requires daily Survival Tests. Modifiers and thresholds from the Survival Test Table (see `survival-test-table` in skills.json).

- Default: Yes
- Skill Group: Outdoors
- Specializations: Desert, Forest, Jungle, Mountain, Polar, Urban, or other terrain

### Swimming (Strength)

Swimming in various bodies of water. Complex Action. Movement per Combat Turn = average of Agility and Strength (in meters). Sprint with Simple Action: `Swimming + Strength [Physical]`, adds 1 m/hit (2 m/hit for elves and trolls). Fatigue applies after extended time (p. 172).

**Holding your breath:** Base time = 60 seconds (20 Combat Turns). To hold breath longer: `Swimming + Willpower` test; each hit extends by 15 seconds (5 Combat Turns). When time expires: 1 box of Stun damage per Combat Turn (unresistable) until character breathes. Stun monitor full = unconscious; then 1 box Physical per Combat Turn.

**Treading water:** Tread for Strength minutes. Then `Swimming + Strength [Physical] (2)` Simple Test. Success = keep treading same duration; failure = 1 box Stun (unresistable). Flotation device doubles time between tests.

- Default: Yes
- Skill Group: Athletics
- Specializations: Dash, Long Distance

### Tracking (Intuition)

Detecting and following trails. Test: `Tracking + Intuition [Mental]` vs. threshold set by terrain (using Success Test Thresholds table, p. 45). If target actively obscures trail, it is an Opposed Test: `Tracking + Intuition` vs. `Sneak + Agility [Physical]`. Net hits provide additional information (trail users, time, etc.). Animal handlers may use Animal Handling as Teamwork.

- Default: Yes
- Skill Group: Outdoors
- Specializations: Desert, Forest, Jungle, Mountain, Polar, Urban, another terrain type

## Social Skills

Social skills are linked to **Charisma** unless otherwise noted. Social tests are Opposed Tests in most cases; see Social Skill Tests table (`social-skill-tests` in skills.json). Social Modifiers table (p. 140) provides dice pool adjustments based on NPC attitude, desired result, and skill-specific context.

When influencing a crowd, the GM uses the crowd leader to oppose; the leader gets +2 dice (ace in the hole) plus a limit bonus equal to the number of people backing their side.

Social skills with language barrier: when using Social skills in a foreign language, you cannot add more dice to your Social pool than your rating in that language.

### Con (Charisma)

Manipulating or fooling an NPC. Covers confidence games and their principles.

- Default: Yes
- Skill Group: Acting
- Specializations: Fast Talking, Seduction

### Etiquette (Charisma)

Social navigation — moving unimpeded through social situations. Acts as a social safety net for blunders. Test: `Etiquette + Charisma [Social]` vs. `Perception + Charisma [Social]`. 3+ net hits improves NPC attitude by one step on the Social Modifiers table.

**Blunders:** If a character blunders socially, Etiquette Test vs. threshold set by GM (severity via Success Test Thresholds, p. 45). Success = character recovers.

**Glitch:** –2 dice penalty to next social test. Critical glitch = NPC attitude moves one step toward Enemy permanently.

- Default: Yes
- Skill Group: Influence
- Specializations: By culture or subculture (Corporate, High Society, Media, Mercenary, Street, Yakuza, etc.)

### Impersonation (Charisma)

Assuming another person's identity, voice, and mannerisms. Limited by character's physical abilities (a dwarf can impersonate a troll over a commlink, but fails in person).

- Default: Yes
- Skill Group: Acting
- Specializations: By metahuman type (Dwarf, Elf, Human, Ork, Troll)

### Instruction (Charisma)

Teaching skills to others. Teacher must have skill at Rating 4+. Teacher can only teach up to their own Instruction Rating. Test: `Instruction + Charisma [Social]`; each hit reduces student's improvement time by 1 day (p. 103).

- Default: Yes
- Skill Group: None
- Specializations: By Active or Knowledge skill category (Combat, Language, Magical, Academic Knowledge, Street Knowledge, etc.)

### Intimidation (Charisma)

Creating the impression of menace. Opposed: `Intimidation + Charisma [Social]` vs. `Charisma + Willpower`. Apply modifiers from Social Modifiers Table (p. 140).

- Default: Yes
- Skill Group: None
- Specializations: Interrogation, Mental, Physical, Torture

### Leadership (Charisma)

Directing and motivating others from a position of authority. Opposed: `Charisma + Leadership` vs. various targets depending on mode. Complex Action for Simple Test.

**Leadership modes:**

- **Command:** Target resists `Leadership + Willpower [Mental]`. Each net hit = target accepts PC as leader for 1 Combat Turn.
- **Direct:** Hits act as Teamwork bonus for one subordinate's Skill or Composure Test.
- **Inspire:** Hits act as Teamwork bonus for subordinates' Surprise Test for the rest of the Combat Turn.
- **Rally:** Subordinates add 1 to their Initiative Score per 2 hits.
- Default: Yes
- Skill Group: Influence
- Specializations: Command, Direct, Inspire, Rally

### Negotiation (Charisma)

Applying charisma, tactics, and situational psychology to achieve better deals.

- Default: Yes
- Skill Group: Influence
- Specializations: Bargaining, Contracts, Diplomacy

### Performance (Charisma)

Executing a performing art (acting, music, comedy, etc.). Simple Test with a Social limit; GM uses hits to gauge artistry. If targeting an audience, they resist with `Charisma + Willpower`.

- Default: Yes
- Skill Group: Acting
- Specializations: By performance art (Presentation, Acting, Comedy, specific Musical Instrument, etc.)

## Magical Skills

Magical skills require the character to be an Aspected Magician, Magician, or Mystic Adept with Magic Rating 1+. See Magic chapter, p. 276.

### Alchemy (Magic)

Creating substances that store spells (charms, preparations, reagents, orichalcum).

- Default: No
- Skill Group: Enchanting
- Specializations: By trigger (Command, Contact, Time), by spell type (Combat Spells, Detection Spells, etc.)

### Arcana (Logic)

Creating magical formulae for spells, foci, and other magical manipulations. Required to understand formulae.

- Default: No
- Skill Group: None
- Specializations: Spell Design, Focus Design, Spirit Formula

### Artificing (Magic)

Crafting magical foci; also used forensically to assess an existing focus's qualities and purpose. See Artificing, p. 306.

- Default: No
- Skill Group: Enchanting
- Specializations: Focus Analysis, Crafting (by focus type)

### Assensing (Intuition)

Reading astral auras and interpreting astral fluctuations. Only awakened characters may take this skill. See Astral Perception, p. 312.

- Default: No
- Skill Group: None
- Specializations: Aura Reading, Astral Signatures, by aura type (Metahumans, Spirits, Foci, Wards, etc.)

### Astral Combat (Willpower)

Fighting in the Astral World. Uses different attributes than physical combat. See Astral Combat, p. 315.

- Default: No
- Skill Group: None
- Specializations: By specific weapon focus type, by opponents (Magicians, Spirits, Mana Barriers, etc.)

### Banishing (Magic)

Disrupting the link between spirits and the physical world; banished spirits return to their native plane and forfeit unfulfilled services.

- Default: No
- Skill Group: Conjuring
- Specializations: By spirit type (Spirits of Air, Spirits of Man, etc.)

### Binding (Magic)

Compelling a summoned spirit to perform additional services. See Binding, p. 300.

- Default: No
- Skill Group: Conjuring
- Specializations: By spirit type (Spirits of Fire, Spirits of Earth, etc.)

### Counterspelling (Magic)

Defending against magical attacks and dispelling sustained magical spells. See Counterspelling, p. 294.

- Default: No
- Skill Group: Sorcery
- Specializations: By spell type (Combat Spells, Detection Spells, etc.)

### Disenchanting (Magic)

Removing enchantments from an item. See Disenchanting, p. 307.

- Default: No
- Skill Group: Enchanting
- Specializations: By type (Alchemical Preparations, Power Foci, etc.)

### Ritual Spellcasting (Magic)

Casting ritual spells. See Ritual Spellcasting, p. 295.

- Default: No
- Skill Group: Sorcery
- Specializations: By keyword (Anchored, Spell, etc.)

### Spellcasting (Magic)

Channeling mana into spells. See Spellcasting, p. 281.

- Default: No
- Skill Group: Sorcery
- Specializations: By spell type (Combat Spells, Detection Spells, etc.)

### Summoning (Magic)

Summoning spirits. See Summoning, p. 300.

- Default: No
- Skill Group: Conjuring
- Specializations: By spirit type (Spirits of Earth, Spirits of Man, etc.)

### Compiling (Resonance)

- Default: No
- Skill Group: Tasking
- Specializations: By sprite type (Data Sprites, Machine Sprites, etc.)

### Decompiling (Resonance)

Deleting previously compiled sprites. See Sprites, p. 254.

- Default: No
- Skill Group: Tasking
- Specializations: By sprite type (Courier Sprites, Fault Sprites, etc.)

### Registering (Resonance)

Registering sprites on the Matrix to convince grids they are legitimate. See Sprites, p. 254.

- Default: No
- Skill Group: Tasking
- Specializations: By sprite type (Crack Sprites, Data Sprites, etc.)

## Technical Skills

Technical skills are used to operate or fix things. Linked attributes vary per skill.

### Aeronautics Mechanic (Logic)

Repairing and maintaining aerospace vehicles. See Building & Repairing, p. 145.

- Default: No
- Skill Group: Engineering
- Specializations: Aerospace, Fixed Wing, LTA (blimp), Rotary Wing, Tilt Wing, Vector Thrust

### Animal Handling (Charisma)

Training, care, riding, and control of non-sentient animals.

- Default: Yes
- Skill Group: None
- Specializations: By animal (Cat, Bird, Hell Hound, Horse, Dolphin, etc.), Herding, Riding, Training

### Armorer (Logic)

Building and maintaining weapons and armor. Requires proper tools and equipment. See Building & Repairing, p. 145.

- Default: Yes
- Skill Group: None
- Specializations: Armor, Artillery, Explosives, Firearms, Melee Weapons, Heavy Weapons, Weapon Accessories

### Artisan (Intuition)

Artistic impression and handcrafting of fine objects. Cannot default.

- Default: No
- Skill Group: None
- Specializations: By discipline (Cooking, Sculpting, Drawing, Carpentry, etc.)

### Automotive Mechanic (Logic)

Repairing ground-based vehicles: commercial automobiles to tanks. Requires proper tools and time. See Building & Repairing, p. 145.

- Default: No
- Skill Group: Engineering
- Specializations: Walker, Hover, Tracked, Wheeled

### Biotechnology (Logic)

Growing organic body parts; basis of cloning and all forms of bioware. Can repair damaged bioware, clone tissue, detect bioware in subject's body. Cannot install or remove bioware.

- Default: No
- Skill Group: None
- Specializations: Bioinformatics, Bioware, Cloning, Gene Therapy, Vat Maintenance

### Chemistry (Logic)

Creating chemical reactions and developing chemical compounds (drugs, perfumes, biopolymers, analysis of unknown chemicals).

- Default: No
- Skill Group: None
- Specializations: Analytical, Biochemistry, Inorganic, Organic, Physical

### Computer (Logic)

Base skill for Matrix interaction and operating Matrix-connected devices. Focuses on understanding operating systems; does not allow code exploitation (Hacking) or slimming mainframes (Hardware). See Using Computer, p. 226.

- Default: Yes
- Skill Group: Electronics
- Specializations: By action (Edit File, Matrix Perception, Matrix Search, etc.)

### Cybercombat (Logic)

Engaging in combat on the Matrix. See Using Cybercombat, p. 226.

- Default: Yes
- Skill Group: Cracking
- Specializations: By target type (Devices, Grids, IC, Personas, Sprites, etc.)

### Cybertechnology (Logic)

Creating, maintaining, and repairing cybernetic parts. Cannot attach/re-attach cybernetics to organic material (not a surgical skill). Used to modify or upgrade cybernetics within cyberlimbs. See Building and Repairing, p. 145.

- Default: No
- Skill Group: Biotech
- Specializations: Bodyware, Cyberlimbs, Headware, Repair

### Demolitions (Logic)

Preparing, planting, detonating, and defusing chemical-based explosives. See Explosives, p. 436.

- Default: Yes
- Skill Group: None
- Specializations: Commercial Explosives, Defusing, Improvised Explosives, Plastic Explosives

### Electronic Warfare (Logic)

Military signals intelligence: encoding, disruption, spoofing, decoding of communication systems. Can manipulate or take over signal of any item's communication system with proper equipment. See Using Electronic Warfare, p. 226.

- Default: No
- Skill Group: Cracking
- Specializations: Communications, Encryption, Jamming, Sensor Operations

### First Aid (Logic)

Emergency medical assistance equivalent to a paramedic. Stabilizes wounds, prevents characters from dying. Cannot perform surgery or repair implants. See Healing, p. 205.

- Default: Yes
- Skill Group: Biotech
- Specializations: By treatment (Gunshot Wounds, Resuscitation, Broken Bones, Burns, etc.)

### Forgery (Logic)

Producing counterfeit items or altering existing items. Test: `Forgery + Logic`; limit is Physical (forged art, signatures, physical objects) or Mental (credsticks, documents, data-based items). Hits = quality of forgery = threshold for Opposed detection test (`Perception + Intuition [Mental]` or `Forgery + Intuition [Mental]`). See Using Forgery, p. 145.

- Default: Yes
- Skill Group: None
- Specializations: Counterfeiting, Credstick Forgery, False ID, Image Doctoring, Paper Forgery

### Hacking (Logic)

Discovering and exploiting security flaws in computers and other electronics. See Using Hacking, p. 226.

- Default: Yes
- Skill Group: Cracking
- Specializations: Devices, Files, Hosts, Personas

### Hardware (Logic)

Building and repairing electronic devices. Requires workspace, materials, and build time. See Building & Repairing, p. 145.

- Default: No
- Skill Group: Electronics
- Specializations: By hardware type (Commlinks, Cyberdecks, Smartguns, etc.)

### Industrial Mechanic (Logic)

Repairing or modifying large-scale machines: assembly line equipment, power generators, HVAC, industrial robots. See Building and Repairing, p. 145.

- Default: No
- Skill Group: Engineering
- Specializations: Electrical Power Systems, Hydraulics, HVAC, Industrial Robotics, Structural, Welding

### Locksmith (Agility)

Building, repairing, and opening mechanical and electronic locks. See Doors, Windows, & Locks, p. 363.

- Default: No
- Skill Group: None
- Specializations: By type (Combination, Keypad, Maglock, Tumbler, Voice Recognition, etc.)

### Medicine (Logic)

Advanced medical procedures: surgeries, long-term disease/illness support, diagnosing medical conditions. Used to implant or remove cybernetics and bioware; cannot repair or maintain implanted devices. See Healing, p. 205.

- Default: No
- Skill Group: Biotech
- Specializations: Cosmetic Surgery, Extended Care, Implant Surgery, Magical Health, Organ Culture, Trauma Surgery

### Nautical Mechanic (Logic)

Maintenance and repair of watercraft. Requires proper equipment and time. See Building & Repairing, p. 145.

- Default: No
- Skill Group: Engineering
- Specializations: Motorboat, Sailboat, Ship, Submarine

### Navigation (Intuition)

Using technology and natural instinct to navigate through territory. Required for unmarked or highly unfamiliar terrain. Test: `Navigation + Intuition [Mental]`. Threshold based on terrain using Success Test Thresholds (p. 45).

- Default: Yes
- Skill Group: Outdoors
- Specializations: Augmented Reality Markers, Celestial, Compass, Maps, GPS

### Software (Logic)

Creating and manipulating programming in the Matrix. Also used by Technomancers for complex forms. See Using Software, p. 226; Threading, p. 251.

- Default: No
- Skill Group: Electronics
- Specializations: Data Bombs, or by complex form (Editor, Resonance Spike, Tattletale, etc.)

### Building & Repairing

Building or repairing is an Extended Test. Gamemaster sets threshold and interval using Task Difficulty Threshold table (p. 48) and Extended Test Thresholds table (p. 48). Modifiers from the Build/Repair Table apply (see `build-repair-table` in skills.json).

> **Cross-reference:** Characters designing new items must have relevant Knowledge skills to conceptualize and create schematics.

## Vehicle Skills

Vehicle skills govern driving and vehicle combat maneuvers. Linked to **Reaction** unless otherwise noted. See Vehicles, p. 198.

### Gunnery (Agility)

Firing any vehicle-mounted weapon regardless of mounting location. Covers manual and sensor-enhanced gunnery.

- Default: Yes
- Skill Group: None
- Specializations: Artillery, Ballistic, Energy, Guided Missile, Rocket

### Pilot Aerospace (Reaction)

All reduced- and zero-gravity aircraft capable of suborbital or extra-orbital flight. Cannot default.

- Default: No
- Skill Group: None
- Specializations: Deep Space, Launch Craft, Remote Operation, Semiballistic, Suborbital

### Pilot Aircraft (Reaction)

Manned or unmanned aircraft operating within planetary atmosphere.

- Default: No
- Skill Group: None
- Specializations: Fixed-Wing, Lighter-Than-Air, Remote Operation, Rotary Wing, Tilt Wing, Vectored Thrust

### Pilot Walker (Reaction)

Any vehicle that walks on two or more legs. Can control the walker physically or remotely.

- Default: No
- Skill Group: None
- Specializations: Biped, Multiped, Quadruped, Remote

### Pilot Exotic Vehicle (Specific) (Reaction)

Must be taken once per specific exotic vehicle. Can control the vehicle remotely. Cannot default. No generalizable specializations.

- Default: No
- Skill Group: None
- Specializations: N/A

### Pilot Ground Craft (Reaction)

All ground-based vehicles excluding legged vehicles. Applies whether pilot is inside or controlling remotely.

- Default: Yes
- Skill Group: None
- Specializations: Bike, Hovercraft, Remote Operation, Tracked, Wheeled

### Pilot Watercraft (Reaction)

Any waterborne vehicle, from inside or via remote control.

- Default: Yes
- Skill Group: None
- Specializations: Hydrofoil, Motorboat, Remote Operation, Sail, Ship, Submarine

### Devising New Active Skills

New Active skills may be introduced by the GM if a clear need exists. The GM assigns the linked attribute, skill group (if any), and whether the skill can be defaulted. New Shadowrun skills should be broad enough to allow multiple specializations. New skills should not be introduced on a per-character basis.

## Knowledge Skills

Knowledge skills represent what the character knows and understands. They do not add dice to Active skill tests directly. Characters receive free Knowledge skill points at character creation based on Logic + Intuition. Four categories exist:

### Street Knowledge (Intuition)

Urban information: movers and shakers, neighborhood dynamics, who to ask and how things get done on the street. Changes rapidly.

Examples: Seattle Street Gangs, UCAS Politics, Ares Macrotechnology, Hong Kong Triads, Security Companies, Sprawl Life.

### Academic Knowledge (Logic)

University-level subjects: history, science, design, technology, magical theory, humanities (cultures, art, philosophy), organizations.

Examples: History, Literature, Economics, Biology, Chemistry.

### Professional Knowledge (Logic)

Normal trades, professions, and occupations: journalism, engineering, business, corporate world information. Useful for run legwork.

Examples: Architecture, Business, Engineering, Military, Police Procedures.

### Interests (Intuition)

Hobbies and personal pursuits. No category limits. What the character does for fun.

Examples: Club Music, Matrix Games, Sports, Street Drugs, Fashion.

### Using Knowledge Skills

Knowledge tests use the Knowledge Skills table as threshold guidelines:

- General Knowledge: 1
- Detailed Knowledge: 2
- Intricate Knowledge: 4
- Obscure Knowledge: 6+

Knowledge skills are not about information freely available on the Matrix. They represent knowledge not everyone knows or can find easily.

### Knowledge Skill Specializations

Knowledge skill specializations narrow a field of knowledge to a specific and limited subset. A specialization of Corporate Culture (Evo) is valid; "Evo Social Culture of the Upper-Echelon Jet Set in the 18-25 Age Bracket" would be too specific. The goal is for the specialization to speak to the character's unique background and remain useful.

## Language Skills

### Language (Intuition)

Ability to converse in a specific language through written and verbal means. Characters speaking multiple languages must purchase a separate skill for each language. No tests are required for everyday conversation; Language tests are called for in critical situations requiring precise translation.

Failing a Language Test: parties are unable to understand each other. A glitch means a meaningful portion of information is misunderstood.

- Default: Yes
- Skill Group: None
- Specializations: Read/Write, Speak, by dialect, by lingo

**Lingos:** Informal languages formed from a mother tongue with heavy slang, jargon, and cultural metaphorical content. Lingos are specializations of base languages (see `2075-lingo-guide` table in skills.json).

**Language Skill Table thresholds:**

- Universal concept (hunger, fear, bodily function): 1
- Basic conversation (concerns of daily life): 1
- Complex subject (special/limited interest topics): 2
- Intricate subjects (almost any technical subject): 3
- Obscure subject (very technical/rare knowledge): 4

**Language modifiers:**

- Speaking a lingo (or variation of a particular language): –2
- Using augmented reality visual display help: +1 to +4

**Social Skills and Language:** When using Social skills in a foreign language, the number of Social skill dice used cannot exceed the character's Language skill rating in that language.

## Using Attributes

When no skill applies, attribute-only tests cover the situation. These pair two attributes to form a dice pool.

### Composure (Charisma + Willpower)

Test to maintain composure when facing emotionally overwhelming situations. Threshold set by severity of the situation (Success Test Thresholds, p. 45). Repeating similar situations eventually eliminates the need for this test (instinct develops).

### Judge Intentions (Charisma + Intuition)

Gauging how much a character can trust another person. Opposed: `Intuition + Charisma` vs. `Willpower + Charisma` of the target. A successful test is a benchmark for gut instinct, not a guarantee of behavior.

### Lifting/Carrying (Body + Strength)

- Base lifting capacity: 15 kg per point of Strength (no test needed).
- Above base: `Strength + Body` test; each hit increases max weight by 15 kg.
- Lifting above the head: base 5 kg per point of Strength; each hit on Lifting Test increases max by 5 kg.
- Carrying weight: Strength × 10 kg with no effort. Additional weight requires a Lifting Test; each hit increases max carry weight by 10 kg.

> **Cross-reference:** Carrying Gear, p. 420.

### Memory (Logic + Willpower)

Recalling information uses the Knowledge Skill Table for thresholds. To memorize deliberately: make `Logic + Willpower` test at time of memorization; each hit adds 1 die to the Recall Test later.

- Glitch: character misremembers some portion of information (e.g., wrong order of digits in a passcode).
- Critical glitch: character completely believes in a false memory of something that never happened.

## Implementation Notes

- The **Active Skill List** (p. 151) is the authoritative reference for which attribute links to which skill. Use this for validation; see `active-skill-list` in skills.json.
- **No-default skills** are italicized in the source book's Active Skill List. Maintain this flag in skill data (e.g., `"canDefault": false`).
- **Skill groups** need special handling for advancement: track whether each group is intact or broken. A broken group should still reference its former group rating on each member skill.
- **Social skill limits**: When building Social skill dice pools, enforce the language cap rule — cap Social dice at Language rating if test is conducted in a non-native language.
- **Magical and Resonance skills** should gate on `Magic >= 1` and `Resonance >= 1` respectively; validate that characters attempting these skills meet the prerequisite attribute.
- **Technical skill Building & Repairing** is an Extended Test framework, not a single roll — ensure the system can track intervals and accumulated hits.
- **Forgery quality** doubles as the detection threshold: store hits from the Forgery test as the threshold opponents must beat to identify the fake.
- **Attribute-only tests** (Composure, Judge Intentions, Lifting/Carrying, Memory) use no skill and should be handled as a special test type that pairs two attributes directly.
- The **Perception + Intuition** Mental test is by far the most frequently called test in play; ensure it has full modifier support from the Perception Test Modifiers table and Environmental Modifiers (p. 173).
- **Lingos** are implemented as Language skill specializations, not separate skills.
- Knowledge skills should support a `category` field with values: `street`, `academic`, `professional`, `interests`.
